# 直播录屏解析方案详细版

## 1. 文件定位

本文档用于指导澜心社数字人直播项目后续如何处理客户提供的完整直播录屏。

它服务于四个目标：

- 为数字人直播间搭建提供可复盘的原始资料结构。
- 从真人直播中蒸馏主播表达、讲课推进、互动承接、促单和异议处理能力。
- 为实时评论回复、人工接管、风险表达拦截和稳定复跑建立资料底座。
- 把目前“凭感觉分类的视频文件夹”转成可以填写、可以抽样、可以复盘、可以沉淀进数字人直播系统的结构化资料。

本文档不是视频解析结果，也不是直播能力验收报告。本轮没有解析真实视频，没有读取、移动、改名、复制、压缩任何视频文件，没有做音频转写，没有做评论 OCR，也没有调用外部 API。

当前状态口径：

- `已确认`：本轮可以先建立直播录屏解析方案和数据填写表。
- `部分成立`：客户的文件夹分类可以作为抽样参考。
- `待验证`：每场直播真实数据、真实成交效果、评论区可见性、声音清晰度、自动解析链路。
- `未知`：凡是没有客户填写、没有录屏证据、没有业务资料证明的数据，一律不补猜。

## 2. 当前资料现状

当前已知资料状态如下：

- `已确认`：用户已有客户提供的完整直播录屏。
- `已确认`：目前缺少每场直播的具体数据。
- `已确认`：客户当前是按主观感受把视频放入不同文件夹。
- `待验证`：每场直播的真实场观、成交金额、成交单数、在线峰值、平均在线、评论数量和投流情况。
- `待验证`：每个视频是否能看到评论区、是否能听清主播声音、是否存在录制中断或画面缺失。

当前文件夹分类包括：

- `今年直播素材`
- `直播较乱`
- `直播效果好`
- `直播效果一般`

这些分类有价值，但价值不是直接证明业务结果，而是帮助后续建立抽样顺序。客户说“效果好”，只能说明客户主观认为这场值得参考；不能直接说明成交好、转化高或适合数字人复刻。客户说“直播较乱”，也不能直接说明视频无价值；它可能正好适合提取负样本、风险表达、节奏崩点和人工接管条件。

因此，本项目后续解析必须把“客户主观分类”和“真实业务数据”拆开记录。

## 3. 核心判断：文件夹分类只能作为弱标签

客户文件夹分类统一定义为弱标签。弱标签可以辅助抽样、排序和人工判断，但不能替代真实数据，也不能直接变成项目结论。

| 文件夹 / 分类 | 建议弱标签 | 可以怎么用 | 不能怎么用 |
| --- | --- | --- | --- |
| `直播效果好` | 正样本候选 | 优先抽样，观察节奏、互动、促单和表达是否完整 | 不能直接写成成交好、转化强、数字人可直接复刻 |
| `直播效果一般` | 中性样本 | 用来提取常规流程、普通互动和可优化点 | 不能直接写成没有价值 |
| `直播较乱` | 负样本 / 风险样本 | 用来提取节奏崩点、冷场、重复、评论失控和接管规则 | 不能直接写成完全不可用 |
| `今年直播素材` | 最新口径样本 | 优先核对课程、权益、价格、主播表达和当前业务口径 | 不能直接写成效果最新最好 |

必须遵守三条硬规则：

1. 不能把“效果好”直接等同于成交好。
2. 不能把“较乱”直接等同于无价值。
3. 没有真实数据时，统一标 `未知 / 待确认 / 待验证`，不得编造。

推荐写法：

- `客户主观标记为效果好，暂列为正样本候选；真实成交数据待验证。`
- `客户主观标记为直播较乱，暂列为风险样本；可用于提取节奏问题和人工接管规则。`
- `该场所属文件夹为今年直播素材，可优先用于核对当前课程口径；直播效果待验证。`

不推荐写法：

- `这场在效果好文件夹，所以成交能力强。`
- `这场较乱，所以不能用。`
- `今年素材代表当前最优直播方式。`

## 4. 客户/用户最小补充信息

客户和用户后续只需要补事实信息，不需要提前完成专业分析。

客户不需要做：

- 不需要做数据清洗。
- 不需要拆解直播节奏。
- 不需要分析成交节点。
- 不需要判断话术属于哪类用户意图。
- 不需要替我方建立话术库、异议库或人工接管规则。

建议每场直播最少补充以下信息：

| 信息 | 填写说明 |
| --- | --- |
| 文件名 | 按原始文件名填写，不改名也可以 |
| 直播日期 | 不知道就填 `未知` |
| 直播账号 | 多账号时必须区分；不知道就填 `待确认` |
| 主播 | 真人主播姓名或代称；不知道就填 `待确认` |
| 主推课程/产品 | 写当场重点讲的课程、服务或产品 |
| 直播主题 | 如讲课、答疑、促单、活动场、混合场 |
| 为什么放在该文件夹 | 写客户自己的原因，例如“互动多”“节奏乱”“今年新话术” |
| 是否有明显异常 | 如声音小、断播、画面卡、评论区看不到、主播离场 |
| 是否有真实数据 | 有就填 `是`，没有就填 `否 / 未知` |

如果有真实数据，再补：

- 场观。
- 成交金额。
- 成交单数。
- 在线峰值。
- 平均在线。
- 评论数量。
- 投流情况。
- 活动节点。

如果没有真实数据，不要留空造成误解。建议填：

- `未知`：现在不知道，但未来可能补。
- `待确认`：需要客户再确认。
- `无`：确定没有这类数据。

## 5. 总体解析路线

直播录屏解析建议分三层走：

1. 全量粗扫。
2. 分层抽样深度解析。
3. 数字人直播能力包沉淀。

三层的关系是：先把所有视频建索引，再选择样本做深度解析，最后才把可复用内容沉淀成数字人直播系统可以使用的能力包。

不要一开始就试图逐帧解析所有录屏。那样容易耗时过长，也容易把没有业务数据支撑的主观判断写成结论。正确做法是先建立总表，把每场直播的来源、弱标签、质量、缺口和解析优先级填清楚。

## 6. 第一层：全量粗扫

全量粗扫的目标不是判断直播好坏，而是建立视频总索引。

每个视频都要至少记录：

- 视频编号。
- 原始文件名。
- 所属文件夹。
- 客户主观标签。
- 客户主观原因。
- 直播日期。
- 直播账号。
- 主播姓名。
- 直播主题。
- 主推课程/产品。
- 直播时长。
- 是否能看到评论区。
- 是否能听清主播声音。
- 是否有明显异常。
- 是否有真实数据。
- 解析优先级。
- 是否进入深度解析。
- 待补资料。

全量粗扫输出：

- `视频总索引`：每个视频一行。
- `可解析清单`：声音、画面、评论区基本可用的视频。
- `优先解析清单`：正样本候选、负样本、今年口径样本。
- `降级参考清单`：声音不清、评论区不可见、只有部分画面的视频。
- `待补资料清单`：缺日期、缺账号、缺课程、缺真实数据、缺客户说明的视频。

全量粗扫的判断口径：

- `已确认`：文件存在、字段已填写、可由用户肉眼确认的事实。
- `待确认`：客户或用户后续需要补的事实。
- `待验证`：需要后续看视频、看数据或做解析才能判断的内容。
- `未知`：当前没有证据。

第一层完成后，至少要知道哪些视频能用、哪些要优先解析、哪些只能作为参考，不能在这一层直接写“数字人直播能力已稳定”。

## 7. 第二层：分层抽样深度解析

深度解析不建议平均分配时间，而是按弱标签分层抽样。

### 7.1 `直播效果好`：正样本候选

建议选 5-8 场进入深度解析。如果视频数量不足，按实际数量选，不补造样本。

重点提取：

- 开场如何留人。
- 主播如何进入课程主题。
- 痛点如何铺垫。
- 案例和信任如何建立。
- 评论如何承接。
- 权益如何讲清。
- 促单节点何时出现。
- 用户犹豫时主播如何处理。
- 哪些表达可以被数字人改写复用。

注意：即使客户标记为效果好，也只能先写“正样本候选”。没有真实成交数据时，不得写“成交能力强”。

### 7.2 `直播效果一般`：中性样本

建议选 3-5 场进入深度解析。

重点提取：

- 常规直播流程。
- 普通场次的节奏密度。
- 哪些话术不差但可以优化。
- 哪些互动没有明显促单效果。
- 数字人复刻时需要补强的地方。

中性样本的价值是建立“普通直播基线”。它帮助我们判断正样本强在哪里、负样本乱在哪里，也帮助后续数字人不只学极端场次。

### 7.3 `直播较乱`：负样本 / 风险样本

建议选 3-5 场进入深度解析。

重点提取：

- 跑题。
- 冷场。
- 重复。
- 评论失控。
- 违规风险。
- 情绪问题。
- 信息不清。
- 成交节奏断裂。
- 技术异常。
- 需要人工接管的条件。

负样本不是废素材。它是数字人系统的“不能怎么做”来源，尤其适合沉淀禁用表达、风险表达、人工接管条件和异常降级规则。

### 7.4 `今年直播素材`：最新口径样本

建议优先用于核对最新业务口径。

重点提取：

- 当前课程 / 产品名称。
- 当前价格、权益和活动表达。
- 当前主播表达习惯。
- 当前用户常问问题。
- 当前异议和顾虑。
- 当前直播间画面和节奏变化。

如果今年素材同时属于效果好、一般或较乱，可以在表格里同时记录“所属文件夹”和“客户主观标签”，但真实效果仍然待验证。

## 8. 第三层：数字人直播能力包沉淀

深度解析完成后，不是把原视频简单归档，而是沉淀成数字人直播系统能使用的能力包。

建议输出以下能力包：

| 能力包 | 作用 | 来源 |
| --- | --- | --- |
| 直播流程时间轴 | 定义开场、讲课、互动、权益、促单、异议、收尾的顺序和时长 | 深度解析记录 |
| 主播话术库 | 沉淀真人主播可复用表达 | 正样本候选和中性样本 |
| 评论问题分类库 | 把评论和问题分成价格、效果、适合人群、课程内容、退款售后等类别 | 评论可见的视频和用户问题资料 |
| 异议处理库 | 建立价格、效果、信任、适合度、售后等回应口径 | 深度解析和业务资料 |
| 促单逼单节点库 | 沉淀限时、名额、福利、价格、案例、行动提醒等促单节点 | 正样本候选 |
| 人工接管规则 | 明确哪些问题不能自动播报，必须人工处理 | 负样本、风险表达、售后/退款/投诉资料 |
| 禁用表达 / 风险表达库 | 标记夸大、绝对化、虚假承诺、效果承诺和合规风险表达 | 负样本和业务规则 |
| 直播复盘标准 | 让后续试播、复跑、问题定位有统一记录方式 | 所有解析表 |

能力包必须保留证据时间戳。后续如果要把某句主播话术变成数字人回复，应能回到来源视频、时间段和当时的评论触发点。

## 9. 无真实数据时的判断口径

无真实数据时，最重要的是不外推。

### 正确写法和错误写法

| 场景 | 错误写法 | 正确写法 |
| --- | --- | --- |
| 客户说效果好 | 这场效果好，所以成交能力强。 | 客户主观标记为效果好，视频中如可观察到节奏完整、互动承接顺、促单节点清晰，可作为正样本候选；真实成交待验证。 |
| 客户说较乱 | 这场较乱，所以不能用。 | 客户主观标记为较乱，可用于提取节奏崩点、风险表达和人工接管规则；是否完全不可用待验证。 |
| 没有成交金额 | 成交金额估计不错。 | 成交金额 `未知`，不得推测；如客户后续补充，再回填。 |
| 评论区看不到 | 这场互动不行。 | 评论区是否可见为 `否 / 待确认`；互动质量不能判断，可降级为主播单向表达参考。 |
| 声音不清 | 话术不可用。 | 主播声音清晰度为 `差 / 待确认`；可先做画面、节奏和异常参考，逐句话术需等待清晰音频或转写。 |

### 字段填写口径

- 不知道的数据填 `未知`。
- 不确定的数据填 `待确认`。
- 已经确认没有的数据填 `无`。
- 需要后续看视频或看真实数据才能判断的内容填 `待验证`。
- 客户主观判断写入“客户主观标签 / 客户主观原因”，不要写入“成交金额 / 成交单数 / 效果结论”。

## 10. 表格填写与解析字段说明

本目录配套 Excel 表格为：

`02_直播录屏数据填写表_live_recording_data_template.xlsx`

表格包含 5 个 Sheet：

| Sheet | 用途 | 何时填写 |
| --- | --- | --- |
| `01_录屏总索引` | 每个视频一行，记录基础事实、弱标签、真实数据缺口和解析优先级 | 用户优先填写 |
| `02_深度解析记录` | 每个关键时间段一行，记录直播阶段、主播表达、评论触发、风险和接管 | 选出样本后填写 |
| `03_用户问题与异议库` | 沉淀评论问题、用户意图、主播回复、推荐数字人回复和接管条件 | 评论可见或用户问题资料补齐后填写 |
| `04_成交与促单节点库` | 沉淀促单、逼单、权益提醒和下钩子节点 | 深度解析正样本候选时填写 |
| `05_负样本与接管规则` | 从较乱场次和风险片段中提取禁用规则、修正建议和人工接管条件 | 深度解析负样本时填写 |

填写顺序建议：

1. 先填 `01_录屏总索引`，把所有视频建档。
2. 根据“所属文件夹 + 客户主观标签 + 资料完整度 + 解析优先级”选择 10-18 场进入深度解析。
3. 再填 `02_深度解析记录`。
4. 从深度解析记录中拆出 `03_用户问题与异议库`、`04_成交与促单节点库` 和 `05_负样本与接管规则`。

用户一开始不需要填满所有 Sheet。第一阶段只要 Sheet 1 足够完整，就能支撑后续抽样。

## 11. 进入下一阶段的验收标准

进入深度解析前，建议满足以下标准：

- 至少完成录屏总索引。
- 每个视频有唯一视频编号。
- 每个视频有所属文件夹。
- 每个视频有客户主观标签。
- 每个视频明确是否有真实数据。
- 每个视频明确是否能看到评论区。
- 每个视频明确是否能听清主播声音。
- 每个视频明确解析优先级。
- 明确哪些视频进入深度解析。
- 不把未知数据编造成真实数据。
- 能选出正样本候选、负样本 / 风险样本、最新口径样本。

进入数字人直播能力包沉淀前，建议满足以下标准：

- 至少有一批正样本候选完成深度解析。
- 至少有一批负样本完成深度解析。
- 至少有今年素材用于核对最新业务口径。
- 每条可复用话术有来源视频和证据时间戳。
- 每条禁用表达或接管规则有来源证据。
- 价格、权益、退款、效果承诺等业务事实已由客户确认，或明确标 `待确认`。

## 12. 风险与失败判定

以下情况不会让资料完全无用，但会改变解析方式：

| 风险 | 影响 | 降级处理 |
| --- | --- | --- |
| 视频声音不可用 | 不能逐句拆话术，不能判断语气和停顿 | 只做画面、流程、异常和时间段参考；等待清晰音频或转写 |
| 评论区不可见 | 不能提取真实评论触发点 | 降级为主播单向表达和流程参考；用户问题库需靠额外资料补 |
| 客户分类原因完全未知 | 弱标签可信度降低 | 在表格里填 `待确认`，先按文件夹做临时分类 |
| 没有课程 / 产品资料支撑 | 价格、权益、退款、适合人群不能自动判断 | 涉及业务事实的内容全部标 `待确认` 或转人工 |
| 没有用户问题 / 异议资料 | 评论回复和异议处理库不完整 | 先从录屏评论区提取；评论不可见时暂缓 |
| 只有短切片，没有完整录屏 | 无法判断完整直播节奏、冷场和长时复跑 | 只能作为话术风格或单点表达参考 |
| 录屏中断或画面缺失 | 时间轴不完整 | 标记异常时间段，不用于完整流程样本 |
| 真实成交数据缺失 | 不能证明转化效果 | 只写促单节点和观察，不写成交有效 |

失败判定：

- 如果没有任何完整录屏，只能做资料结构和字段模板，不能做直播节奏解析。
- 如果评论区全部不可见，不能建立可靠的实时评论回复规则。
- 如果业务资料缺失，AI 不能自动回答价格、权益、退款、效果承诺。
- 如果没有人工接管规则，不能进入稳定试播。
- 如果把客户主观标签写成真实成交结论，本阶段资料结构失败，需要回滚修改。

## 13. 下一步建议

建议下一步按以下顺序执行：

1. 用户先填写 Excel 的 `01_录屏总索引`。
2. 每个视频至少补齐原始文件名、所属文件夹、客户主观标签、客户主观原因、直播日期、直播账号、主播、直播主题、主推课程/产品、是否有真实数据、解析优先级。
3. 如果真实数据暂时没有，统一写 `未知 / 待确认 / 无`，不要空着，也不要估算。
4. 从 `直播效果好` 中选 5-8 场正样本候选。
5. 从 `直播较乱` 中选 3-5 场负样本 / 风险样本。
6. 从 `今年直播素材` 中选最新口径样本，优先核对课程、权益、价格和主播表达变化。
7. 总计选择 10-18 场进入深度解析；数量不足时按实际数量调整，不编造。
8. 深度解析后再建立主播话术库、异议库、促单库、禁用表达库和人工接管规则。
9. 最后再进入数字人直播大脑适配，把可复用话术、评论分类、风险边界和接管规则变成可测试的规则文件或知识库。

本阶段的正确目标不是证明数字人直播能力已经稳定，而是把客户已有录屏整理成可信、可查、可复盘、可继续解析的数据结构。
