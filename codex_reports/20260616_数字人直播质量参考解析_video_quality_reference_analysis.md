## 不能直接对客户承诺什么

`待验证`：当前不能承诺“普通 2D 或普通 3D 就能完全还原真人直播”。也不能承诺“真人还原级直播已经可稳定交付”。当前只能说：参考质量目标偏 `L4 真人还原级`，需要高成本神经数字人/真人克隆/混合方案，并且必须先做 15 秒最小样片 probe，再决定是否进入 4-5 小时稳定直播验证。

## 最小下一步

`通用建议`：先让供应商或内部候选路线做 15 秒样片，不要先谈完整直播间承诺。样片必须覆盖快速口播、停顿、眨眼、嘴角变化、眼神转移、轻微转头、半身/近景、至少一次手势或商品遮挡。只要出现嘴型错位、眼神死、脸僵、眨眼怪、手部/道具穿帮、皮肤蜡感，即判定不能承诺真人还原级。

# 参考视频真人质量画像

## 总体画像

`部分成立`：抽帧观察显示，3 个候选视频均是竖屏直播录屏或直播切片形态，带平台 UI、评论区、商品卡、活动标签和主播口播。画面中的人物表现包含商品展示、手势、头肩移动、眼神转移、眨眼候选、嘴型变化和直播式节奏。

`待验证`：仅凭本地视频不能确认视频来源一定是真人、是否有滤镜、是否有后期处理、是否经过平台转码或 AI 修饰。报告只把“可观察到的真人直播级画面特征”作为参考画像，不把“视频一定是真人原始直播”写成已确认。

## V1 质量画像

V1 是服装店/货架场景，主播在半身到全身之间移动，展示上衣、黑色服装、手机屏幕和店内背景。可观察点：

- 约 0-12s：主播半身/全身出镜，背景为服装货架，直播 UI 完整，口播状态明显。
- 约 18-30s：主播拿起黑色衣物，手臂、衣物和脸部有遮挡关系。
- 约 30-42s：手机靠近镜头展示穿搭图片，前景手机、背景人像、画面焦点关系复杂。
- 约 48-60s：主播从侧身/半身转回正面，有头部转动、肩颈变化和手势。
- 约 66-74s：主播回到更远半身/全身，动作幅度更接近真实直播走位。

质量特征：动作空间大、道具多、遮挡多、镜头层次明显，是普通 2D 数字人最难模拟的一类参考。

## V2 质量画像

V2 是鞋类近景商品讲解。主播脸部占画面比例更大，频繁拿鞋靠近镜头，展示鞋面、鞋底和商品卡。可观察点：

- 约 0-8s：主播近景拿鞋，嘴型持续变化，眼神在镜头和商品之间切换。
- 约 8-16s：鞋底靠近镜头，商品局部遮挡脸部边缘，手部和商品细节非常突出。
- 约 16-24s：主播把鞋移回胸前，出现手势、微笑、眼神变化和快速讲解状态。
- 约 28-35s：两只鞋/黑色鞋切换，前景道具运动幅度大。

质量特征：脸部细节最清楚，最适合检查眼神、嘴角、牙齿和眨眼；同时手和商品入镜很重，对数字人生成路线压力很大。

## V3 质量画像

V3 是服装/穿搭直播切片，分辨率和码率低于 V1/V2，平台压缩更明显。主播全身或半身站在固定背景前讲解，手部动作较小但持续存在。可观察点：

- 约 0-18s：主播全身/半身站立，双手在身前变化，口播连续。
- 约 24-42s：出现近景脸部、头部转向和手部抬起。
- 约 48-60s：脸部更近，嘴型变化明显，但画质压缩和柔化更重。
- 约 66-74s：回到全身/半身，动作相对 V1/V2 更模板化。

质量特征：更接近“可播直播切片”而非最高清真人质感。它对 2D/3D 路线的压力低于 V1/V2，但仍要求稳定口型、眼神和半身动作。

# 直播真实感分析

## 可观察真人直播感

`部分成立`：3 个视频都具有强直播间特征：平台 UI、实时评论、商品卡、活动标签、主播边讲边展示商品、镜头前后景变化、手和商品遮挡、商品上架/价格/促销信息同步出现。

这些特征对数字人路线意味着：

1. 只生成一个好看的脸不够。
2. 只把文案念出来不够。
3. 只做离线 15 秒视频不等于可做实时直播。
4. 直播间真实感来自“人、货、场、话术、互动节奏、UI 证据、突发响应”的组合。

## 皮肤、头发、牙齿、光影

`部分成立`：V1/V2 画质较高，脸部有明显美颜/柔化，皮肤平滑、眼睛高亮、口红/牙齿边缘清楚。V3 压缩更明显，脸部细节弱。当前参考的“真人感”并非 raw camera 真实质感，而是短视频/直播平台美颜后的真人直播感。

对数字人路线的含义：

- 2D 路线容易皮肤蜡感、眼神空、牙齿/嘴角不稳。
- 3D 路线容易材质假、头发假、光照与真实店铺环境不一致。
- 神经数字人路线更容易保持真人脸部统计特征，但仍可能在牙齿、舌头、眼神和侧脸处露馅。

# 2D数字人路线可行性

## A. `2D 图片免训 / 图片驱动数字人`

`不建议承诺` 当前参考级质量。

能达到的部分：

- 低成本正脸口播。
- 轻微头部摆动。
- 基础眨眼和嘴巴开合。
- 固定背景或轻量直播画面。

做不到或风险高的部分：

- V1/V2 中大量手部、衣物、鞋、手机、商品遮挡。
- 远近景切换和真实相机透视关系。
- 眼神焦点随商品/评论/镜头变化。
- 快速口播下的自然嘴角、牙齿、舌头和闭口音。
- 4-5 小时连续直播中的表情随机性和稳定性。

适配判断：

- 可作为 `L1 可做低成本数字人，但不像真人`。
- 如要用于项目初期内部演示，必须限定为“正脸半身、少手势、无真实商品近景遮挡、短时样片”。
- 不适合对客户承诺“完全还原真人直播”。

## B. `2D 视频训练数字人 / 真人克隆数字人`

`部分成立`：如果供应商具备高质量训练、足够授权素材、实时驱动和人审流程，可能达到局部 `L2-L3`。

能达到的部分：

- 比图片免训更像本人。
- 近景脸部、表情、口型自然度有机会提升。
- 可复制主播语气、风格和部分微表情。
- 更适合 V2 这类近景商品讲解样片。

做不到或风险高的部分：

- 手拿商品、鞋底近镜头、衣物遮挡脸部等复杂交互仍很难。
- 如果只是头部/半身模型，V1 的全身走位和手机展示会明显穿帮。
- 实时互动时，语音生成、口型驱动、表情和身体动作可能不同步。
- 长时间直播容易出现重复动作、眨眼节奏机械、眼神空。

适配判断：

- 可作为第一轮 15 秒样片 probe 路线。
- 必须要求供应商提供“带商品遮挡的近景口播”样片，而不是只给棚拍正脸 demo。
- 不能在样片前承诺真人还原级。

# 3D数字人路线可行性

`部分成立`：3D 数字人适合实时控制、动作编排、稳定推流和可交互系统，但不天然适合本轮参考的“真人直播级皮肤/眼神/微表情”。

能达到的部分：

- 实时驱动更可控。
- 动作、手势、站位、转身可以通过 rig / motion capture / animation 编排。
- 适合长时间直播稳定执行，便于和 OBS/直播推流/问答系统集成。
- 可避免部分 2D 头像在大幅转身或身体动作上的限制。

做不到或风险高的部分：

- 高写实 3D 的皮肤、头发、牙齿、眼神、嘴角非常容易出现假感。
- 商品真实纹理、鞋底近镜头、衣物软体变化和手指接触很难自然。
- 要达到 V1/V2 的美颜真人直播质感，需要高成本扫描、绑定、材质、灯光、动作捕捉和实时渲染调优。
- 3D 稳定不等于真人像；它更可能是“可播数字主播”，不是“像真实主播本人”。

适配判断：

- 可用于 `L2 可做可播数字人，但微表情和口型需人审`。
- 若客户接受风格化或轻写实，3D 是稳定性路线。
- 若客户目标是“看起来就是参考视频这种真人直播”，普通 3D 不够。

# 真人克隆 / 神经数字人 / 混合方案可行性

`通用建议`：当前参考目标更接近 `真人克隆 / 神经数字人 / 混合方案`，而不是普通 2D 或普通 3D。

推荐混合结构：

1. 真人授权素材：照片、自然说话视频、直播口播视频、不同角度、不同表情、不同语速。
2. 神经数字人脸部/半身：负责脸、眼神、嘴型、微表情和主播身份一致性。
3. 语音克隆或真人音色 TTS：负责语气、停顿、促单节奏。
4. 实时驱动：接入脚本、评论意图、商品知识库、人工接管。
5. 混合商品层：真实商品图/视频/OBS 画面叠加，避免完全生成手拿商品。
6. 人工中控：高风险评论、价格/权益/退款/效果承诺必须可接管。

能达到的部分：

- 最有机会接近参考视频中的真人脸部和口播质感。
- 可在短样片中验证眼神、口型、微表情和直播口吻。
- 可把“人像生成”和“商品展示/直播 UI”拆开，降低道具穿帮。

仍然高风险的部分：

- 实时直播延迟和长时间稳定性。
- 快速语音下的口型精度。
- 手部和商品遮挡。
- 多角度转头、侧脸、低头看商品、抬头看评论。
- 平台审核、授权、肖像/声音合规。
- 供应商 demo 与真实业务直播素材之间的落差。

适配判断：

- 若目标是 `L3 可接近真人直播`，必须走高质量训练和样片验证。
- 若目标是 `L4 真人还原级`，必须走高成本真人克隆/神经数字人/专业供应商深度定制，且仍需样片验证，不可直接承诺。

# 最小样片验证方案

## 样片规格

- 时长：15 秒。
- 画幅：竖屏，至少 720p；如果供应商声称可达参考级，优先 1080p 或接近 V1/V2 的高清竖屏。
- 镜头：半身或近景脸部，不允许只给远景。
- 文案：真实直播促单口播，包含商品名、利益点、价格/权益提醒、一次停顿、一次快速语音。
- 动作：自然眨眼、眼神轻微转移、头部轻微转动、嘴角变化、肩颈变化、至少一次手势。
- 商品：至少一版带真实商品或商品图层；如供应商不能处理手拿商品，必须明确采用混合商品层方案。
- 音频：必须有音频，不能交静音样片。

## 必须出现

1. 眨眼：自然，不突兀，不固定节奏。
2. 停顿：停顿时嘴闭合自然，眼神不死。
3. 快速语音：至少 3 秒较快促单话术，检查嘴型。
4. 嘴角变化：微笑/强调时嘴角、牙齿、脸颊联动。
5. 头部轻转：不能只正脸锁死。
6. 自然眼神：能看镜头，也能轻微看商品/评论方向。
7. 商品/手势：至少一次手势或商品遮挡，不允许完全避开难点。

## 验收项

| 验收项 | 通过标准 | 失败判定 |
|---|---|---|
| 口型 | 快速语音、闭口音、停顿基本对齐 | 嘴型错位、音频在说但嘴没动、嘴动但无声 |
| 眨眼 | 频率和速度自然 | 眨眼怪、眼睑穿帮、固定周期 |
| 眼神 | 有焦点，有轻微变化 | 眼神死、飘、左右不一致 |
| 微表情 | 嘴角、脸颊、眉眼联动 | 脸僵、只有嘴动 |
| 动作 | 头肩和手势自然 | 动作模板化、抖动、穿模 |
| 商品 | 商品层级合理 | 手指变形、商品融脸、遮挡穿帮 |
| 清晰度 | 皮肤、牙齿、头发不过分糊 | 蜡感、糊脸、牙齿变形 |
| 直播感 | 像直播口播，不像单张图念稿 | 机械播报、镜头无生命感 |

## 通过后的下一步

15 秒样片通过只代表 `部分成立`，下一步还要做：

1. 60 秒连续口播样片。
2. 5 分钟脚本 + 评论插入模拟。
3. 20-30 分钟链路试播。
4. 4-5 小时人工中控兜底直播验证。
