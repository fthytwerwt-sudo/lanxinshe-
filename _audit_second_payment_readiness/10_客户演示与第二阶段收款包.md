# 客户演示与第二阶段收款包

## 演示前提

当前状态为 `NO`，本文件是补齐 P0 后的客户演示方案，不表示现在已经可以验收或收款。

## 演示流程（建议 20–25 分钟）

1. **边界说明（2 分钟）**：本次展示第二阶段受控 V0，不展示真实平台长时间开播。
2. **三类样片（4 分钟）**：逐一播放讲解型、回复型、促销型约 30 秒最终文件，显示文件名、模型/声音/视觉版本和 fallback 标记。
3. **知识与规则（3 分钟）**：展示已确认知识条目及六类规则版本，不展示未经复核的 19,424 chunks 作为已完成知识库。
4. **评论一：知识咨询（3 分钟）**：历史评论回放 → 分类 → 知识引用 → 动态回复 → 阿里 3D 播报 → 日志。
5. **评论二：促销/异议（3 分钟）**：展示规则约束和非固定回复。
6. **评论三：高风险（3 分钟）**：展示禁限表达拦截和 manual takeover，不强行自动回答。
7. **可重复性（2 分钟）**：再次执行三条，核对不同 event_id、结构一致和日志完整。
8. **限制与下一阶段（2 分钟）**：说明真实平台 API、20–30 分钟试播、4–5 小时稳定性在后续阶段。
9. **逐项验收（3 分钟）**：现场勾选验收表并确认书面回执方式。

## 第二阶段交付清单

- 讲解型约 30 秒最终样片及生成/版本说明。
- 回复型约 30 秒最终样片及生成/版本说明。
- 促销型约 30 秒最终样片及生成/版本说明。
- 声音形象版本、视觉形象版本和客户确认记录。
- 人工复核的最小知识库清单。
- 评论分类、回复、异议、促销、禁限表达、人工接管规则。
- V0 启动说明、三评论输入、预期结果、完整日志和失败记录。
- 阿里 3D 播报回执或经双方书面同意的替代验收说明。
- 已知限制、第三阶段边界、验收表和正式书面确认。

## 验收确认项

1. 三类样片身份明确、可播放、声音/视觉版本一致。
2. 三条不同评论确实触发不同处理路径，不是固定模板串演。
3. 知识回答可显示来源；无来源或高风险会转人工。
4. 阿里 3D 表现层能接收最终回复并返回可审计回执。
5. 日志可回看输入、分类、知识/规则、回复、风险、转人工/播报。
6. 客户理解真实平台接入和长时间试播不属于本次验收。
7. 验收结论、例外和后续事项形成书面记录。

## 客户常见质疑

| 质疑 | 建议回答 |
|---|---|
| “这是不是提前写好的固定回复？” | 现场用三条不同评论并随机更换一条已复核评论，展示 event log、知识引用和模型输出。 |
| “为什么不是直接在抖音开播？” | 本阶段验收受控 V0；平台接入、OBS/RTMP 和长时间试播属于下一阶段。 |
| “遇到不能回答的问题怎么办？” | 展示 fail-closed、禁限表达和人工接管，不把风险转嫁给模型。 |
| “视频样片和实时数字人是一回事吗？” | 不是。离线样片证明视觉/声音方向；V0 另用表现层适配与播报回执证明动态链路。 |
| “知识会不会乱说？” | 只使用已复核条目并显示 source_id；无来源/冲突转人工。 |
| “为什么现在要付款？” | 只有合同生效、第二阶段清单逐项完成并获得书面验收后，才按约定节点主张付款。 |

## 演示失败时的降级方案

1. 保留失败现场和 event log，不删改证据。
2. 阿里 3D 失败时切换 `manual_takeover` 或预先声明的 fake adapter，只证明上游逻辑；明确标记“不构成 3D 验收”。
3. Qwen/知识检索失败时展示规则型安全回复或转人工；不得使用提前录制视频冒充实时结果。
4. 样片播放失败时只恢复到已校验原文件，不做现场本地修复。
5. 中止正式验收，形成缺陷清单和复演日期；不要求客户在证据不完整时确认通过。
