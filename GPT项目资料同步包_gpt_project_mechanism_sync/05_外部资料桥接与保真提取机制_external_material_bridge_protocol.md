# 外部资料桥接与保真提取机制

## 文件定位

本文件规定外部资料、参考内容、用户转述、Web 检索结果进入当前项目时，如何先保真提取，再桥接执行。

本文件只承载资料处理机制，不承载外部资料正文或项目事实。

## 触发条件

- 用户带回网页、截图、视频、音频、文案、竞品、研究摘要。
- Web 输出了资料。
- GPT 准备把外部资料写进项目知识或下发给 Codex。
- Codex 需要根据外部资料执行文件任务。

## 默认四层

1. 原始材料：用户给了什么、来源是什么。
2. 保真事实：材料中确实可观察到什么。
3. 执行桥接：能转成哪些字段、组件、约束、验收标准。
4. 推测判断：AI 的解释和建议，必须标 `推测`。

## 可进入项目知识的内容

- 来源清楚的稳定规则。
- 可核查资料。
- 抽象后的方法论。
- 已通过当前项目事实源验证的内容。

## 只能作为临时参考的内容

- 没有来源的摘要。
- 未核实的经验判断。
- 平台 UI、品牌资产、可识别第三方素材。
- 一次性搜索结论。
- 与当前 GitHub `main` 文件冲突的资料。

## 禁止行为

- 把外部资料直接写成项目事实。
- 把用户转述写成已验证结论。
- 复制第三方 UI、品牌资产或可识别素材。
- 只做长摘要，不输出执行字段。
- 用“像它一样”替代具体可执行规则。

## Codex 前置条件

Codex 只能读取明确提供在本轮输入、仓库文件或允许资料路径中的内容。

如果资料没有写入本轮输入或项目事实源，Codex 不得凭记忆补全。

## 一句话执行口径

外部资料先保真，再桥接；可以学方法，不能把参考材料偷换成当前项目事实。
