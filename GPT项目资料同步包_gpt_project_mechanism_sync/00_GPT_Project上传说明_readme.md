# GPT Project 上传说明

## 文件定位

`已确认`：本目录是给 GPT Project（当前项目配合机制层）上传 / 读取的机制包。

本包只承载协作方式、判断顺序、事实源边界、Codex（本地执行、验证、落库层）下发规则和质量保障机制，不承载项目事实。

## 三层 + 执行层

- 账号记忆：跨项目长期协作偏好。
- GPT Project（当前项目配合机制层）：保存配合机制。
- GitHub `main`（当前项目事实源）：保存项目事实。
- Codex（本地执行、验证、落库层）：执行、验证、commit（提交）、push（推送）、remote HEAD（远端最新提交指针）验证。

## 使用方式

将本目录内文件上传到 GPT Project，作为当前项目的配合机制层。GPT Project 读取本包后，应知道：

- 什么时候先判断，什么时候直接执行。
- 什么时候必须拦截、澄清或 blocked。
- 如何根据 GitHub `main` 读取项目事实。
- 如何把 Codex 执行单写到可执行、可验证、可回滚。
- 如何防止旧项目事实、聊天中间态、外部资料污染当前项目事实。

## AGENTS.md 关系

`AGENTS.md`（仓库接手入口文件）属于 GitHub `main` 当前项目事实源，不直接放入 GPT Project 上传包。

GPT Project 上传包只同步可复用配合机制；仓库入口规则、当前项目事实、执行日志、阶段状态和验证结果必须回 GitHub `main` 当前文件读取。

## 禁止上传内容

本包不得包含：

- `.env`、token、API key、账号密码、cookie。
- 图片、视频、音频、zip、缓存、运行输出。
- 旧项目业务身份、素材路径、API 状态、验收结论、执行日志或能力结论。
- 当前项目阶段结果、客户需求细节、执行日志、验收结论。
- GitHub 当前事实文件的复制件。

## 冲突处理

- 用户本轮输入和本包冲突：以用户本轮输入为准。
- 本包机制和 GitHub 事实冲突：机制归本包，事实归 GitHub。
- 旧项目经验和当前项目冲突：旧项目只作为 `P2` 参考，不得覆盖当前事实。
- 外部资料和当前事实冲突：外部资料先标 `待验证` 或 `推测`，不得直接进入项目事实层。

## 一句话执行口径

GPT Project 存配合机制，GitHub `main` 存项目事实，Codex 负责执行落库；机制可复用，事实必须回到当前仓库验证。
