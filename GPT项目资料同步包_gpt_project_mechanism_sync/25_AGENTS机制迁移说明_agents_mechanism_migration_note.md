# AGENTS 机制迁移说明

## 文件定位

本文件记录一次只迁移协作机制、不迁移旧项目事实的 AGENTS 机制提取结果。

本文件只承载迁移说明和边界，不承载当前项目事实，也不承载旧项目事实。

## 读取来源

- 旧直播切片项目的 `AGENTS.md`（仓库接手入口文件）。
- 旧直播切片项目的 GPT Project 配合机制同步包。

上述来源是 `P2` 机制参考，不是当前项目事实源。

## 已迁移机制

### 1. 接手入口机制

已迁移为：GPT / Codex 接手仓库任务时，先确认 workspace（当前工作目录 / 工作区）、remote（Git 远端仓库地址）、branch（分支）、当前用户输入和仓库入口文件。

### 2. 读取顺序机制

已迁移为：先读用户本轮输入，再读当前仓库 `AGENTS.md`，再读 GPT Project 机制包，再读项目事实，再读执行报告和验证记录。

### 3. GPT Project 上传包纪律

已迁移为：GPT Project 只放配合机制，不放项目事实；`AGENTS.md`、执行日志、项目事实文件必须回 GitHub `main` 当前文件读取，不复制进 GPT Project 上传包。

### 4. Codex push 纪律

已迁移为：凡产生仓库文件改动，`completed` 必须等到 commit（提交）已创建、push（推送）已成功、remote HEAD（远端最新提交指针）已验证。push 失败必须标 `blocked_push_failed`。

### 5. path-limited stage 纪律

已迁移为：禁止 `git add .`，必须只 stage 本轮相关文件。

### 6. 文件命名规则

已迁移为：Codex 新建自定义文件和目录默认使用“中文 + English slug”命名；工具强制约定文件名可保留固定原名。

### 7. 状态标记规则

已迁移为：只使用受控状态词表达判断强度。

### 8. 真实意图澄清闸门

已迁移为：方向型、能力型、demo 型、项目生死判断型或 Codex 执行型任务，GPT 必须先澄清真实目标、成功标准、失败标准和停止条件。

### 9. 实现设计层闸门

已迁移为：涉及直播、自动化、脚本、工具路线、机制修改或 Codex 执行的任务，必须先补实现设计层。缺实现设计层必须标：

```text
blocked_need_implementation_design_layer
```

### 10. 禁止迁移旧项目事实规则

已迁移为：旧项目只能作为机制来源，不能作为当前项目事实来源。

## 明确排除内容

以下内容已明确排除，不进入当前项目机制包：

- 旧项目身份。
- 旧项目具体目录信息。
- 旧项目 GitHub 仓库名。
- 旧项目能力状态。
- 旧项目素材路径。
- 旧项目任务状态。
- 旧项目执行日志。
- 旧项目业务身份和业务方向。
- 旧项目 API 状态。
- 旧项目验收结论。

## 当前项目适配说明

- 所有机制已改写为当前项目可用的通用配合机制。
- 本文件不承载当前项目事实。
- 本文件不承载旧项目事实。
- 当前项目事实仍只能从 GitHub `main` 当前文件和用户本轮明确输入读取。
- 旧项目 AGENTS.md 只作为 `P2` 机制参考，不得覆盖当前项目 `P0 / P1`。

## 一句话执行口径

AGENTS 可以提取协作纪律，不能迁移项目事实；机制进入 GPT Project，事实回到 GitHub `main`。
