# 真实意图澄清闸门机制

## 文件定位

本文件规定 GPT 在下发 Codex 前，如何确认用户真实目标、验收标准、失败判定和停止条件。

本文件只承载澄清机制，不承载项目事实。

## 触发条件

- 用户说“让 Codex 做一下 / 给我 Codex prompt / 直接执行”。
- 用户问“能不能做出来 / 值不值得继续 / 方向是不是错了”。
- 输入里同时有方向判断和执行请求。
- 任务影响项目主线、工具选型、机制设计或继续投入。
- 用户表达“不对 / 别抽卡 / 别浪费时间 / 差点意思”。

## GPT 默认先回答

- 本轮真正要判断什么。
- 本轮不判断什么。
- 成功标准。
- 失败标准。
- 停止条件。
- 默认假设。
- 待确认位。
- 是否需要进入六层需求确认。

## 可以直接执行的条件

只有以下内容清楚，才可以下发 Codex：

- 目标清楚。
- 边界清楚。
- 实现设计清楚。
- primary_route 清楚。
- fallback 清楚。
- capability boundary 清楚。
- 是否需要 probe 清楚。
- 验收清楚。
- 是否需要 commit / push 清楚。
- 是否允许外部 API、安装依赖、读取敏感文件清楚。

## blocked 条件

- 真实目标不清。
- 成功 / 失败 / 停止条件不清。
- 缺实现设计层。
- 不知道是否允许扩大影响面。
- 不知道外部资料是否可用。
- 不知道哪些是项目事实、哪些只是参考。

## Codex 收到缺失执行单时

Codex 应回报：

```text
blocked_missing_true_intent_gate
```

或：

```text
blocked_need_implementation_design_layer
```

## 一句话执行口径

先聊清真实目标，再下发执行；不清楚目标的 Codex prompt，不进入落库执行。
