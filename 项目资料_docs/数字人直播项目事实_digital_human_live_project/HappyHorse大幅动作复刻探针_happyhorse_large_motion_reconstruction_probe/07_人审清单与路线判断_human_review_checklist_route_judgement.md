# 人审清单与路线判断

状态：`route_judgement_pending_human_review`

## 用户人审清单

请按 3 条最终候选逐条看：

- 身体动作节奏是否接近源视频。
- 手势方向和幅度是否接近。
- 头肩节奏是否接近。
- 表情变化是否像源视频。
- 眨眼 / 视线是否自然。
- 口型是否与音频同步。
- 人物身份和脸是否稳定。
- 大动作下是否僵硬、漂移、崩坏。
- 背景和衣服是否保持，不要被模型随意改。

## 路线判断

- HappyHorse 是否明显优于 Wan-S2V 当前路线：`部分成立`。对动作时间线保留明显更强，但路线性质是 `video-edit`，不是单图数字人 motion transfer。
- 是否建议继续 HappyHorse：`通用建议`，建议先人审本轮 3 条最终候选。
- 是否进入 30 秒 / 60 秒：`待验证`，必须用户人审确认 15 秒观感后再做。
- 是否需要补充更大动作样本：`通用建议 yes`。当前 3 条严格来说不是走动/转身/站坐级别大位移，下一轮应补真正全身大动作。

## 下一轮建议

1. 如果用户认可本轮 15 秒动作/表情保留，下一轮测 30 秒稳定性。
2. 如果动作好但口型差，下一轮单独测 lipsync。
3. 如果表情好但身体差，补更大身体动作样本或继续找 motion reference 专用模型。
4. 如果要做“新数字人身份复刻源动作”，需要验证 HappyHorse 是否支持目标身份图 + motion source 的明确工作流。
