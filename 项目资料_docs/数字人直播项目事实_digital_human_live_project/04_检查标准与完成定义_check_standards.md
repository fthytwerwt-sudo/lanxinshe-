# 检查标准与完成定义

## 文件定位

本文件记录当前项目检查标准和完成定义入口。当前仅写通用项目事实入口，不写未验证能力结论。

## 已确认

- `completed` 必须同时满足：文件已创建或修改、验证已执行、commit（提交）已创建、push（推送）成功、remote HEAD（远端最新提交指针）已验证、local HEAD（本地最新提交指针）等于 remote HEAD。
- 未经验证的 API 状态、数字人能力、直播间能力、评论回复能力、成交能力不得写成 `已确认`。
- 技术验证通过不等于用户人审通过，也不等于业务目标通过。

## 待补全

- 数字人直播间搭建检查标准。
- 数字人主播角色 / 形象 / 声音检查标准。
- 直播脚本与流程检查标准。
- 课程产品讲解检查标准。
- 评论识别和回复检查标准。
- 促单、逼单、异议处理检查标准。
- 对标直播间解析检查标准。
- 直播录屏解析与复盘检查标准。
- AI 与人工兜底分工检查标准。

## 禁止完成定义

- 本地文件存在但未 push，不得写 `completed`。
- push 成功但 remote HEAD 未验证，不得写 `completed`。
- API 未 probe，不得写 API 可用。
- 未经用户人审，不得写审美或业务通过。
