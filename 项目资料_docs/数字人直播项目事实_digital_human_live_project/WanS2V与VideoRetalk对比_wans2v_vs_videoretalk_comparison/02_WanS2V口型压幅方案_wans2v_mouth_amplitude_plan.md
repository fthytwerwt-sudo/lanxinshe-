# 02_WanS2V 口型压幅方案

状态：`completed_plan_only_no_generation`

## 主结论

`已确认`：Wan-S2V 原始输出主要卡在 `mouth_face_ratio_p90` 偏大；本轮不继续生成，不做本地嘴部压缩，不把后处理作为默认路线。

`primary_route`：下一轮只做模型侧、输入侧、文案节奏侧、参考帧/姿态侧、生成参数/prompt 侧的最小 probe。

`禁止默认路线`：本地嘴部压缩、局部修嘴、羽化融合、换音轨、补帧不作为默认路线。

## 方案一：模型侧控制

目标：让 Wan-S2V 原生输出降低嘴巴相对脸高的开合比例，优先压 `mouth_face_ratio_p90` 到 `<=0.075`。

做法：

- 继续使用 `wan2.2-s2v`，但将 prompt 明确写成低幅度口播。
- 正向约束：`small mouth movement`、`subtle lip motion`、`natural closed-mouth speaking`、`minimal jaw opening`。
- 负向约束：`wide open mouth`、`exaggerated mouth movement`、`teeth exposed`、`large jaw motion`。
- 同一输入只跑 2-3 个最小候选，不进入长视频扩展。

为什么能压口型：当前原始输出不是连续大面积失败，而是 mouth-to-face 比例超线；模型侧减少 jaw opening 和 teeth exposure 有机会降低嘴部纵向幅度。

风险：

- Wan-S2V 是否真正响应 mouth amplitude prompt 仍 `待验证`。
- 压得过强可能导致闭嘴说话、同步变差或表情僵硬。

验收指标：

- `mouth_face_ratio_p90 <= 0.075`
- `p90_mouth_open_ratio <= 0.32`
- `fail_frame_percent <= 15`
- `longest_fail_streak < 5`
- 用户人审不觉得“一眼假”或嘴部僵硬。

blocked 条件：

- 模型不支持相关参数或 prompt 控制无效。
- 需要本地后处理才能过线。
- 新候选出现脸漂、五官变形或明显不同步。

## 方案二：输入音频侧控制

目标：通过输入声音的响度、语速、停顿和爆破音控制，减少模型被高能量音素驱动出大开口。

做法：

- 保持目标声音来源，但生成低爆口版本音频。
- 控制语速在稳定区间，不追求极短时长硬塞。
- 降低尖峰和爆破音，保留自然停顿。
- 对比两种音频：正常版、轻柔版；不换音轨修最终结果，只作为下一轮模型输入。

为什么能压口型：S2V 通常会受音频能量、音素密度和节奏影响；高能量、密集、爆破音多的输入更容易带来夸张开口。

风险：

- 声音太轻可能影响直播感染力。
- 语速太慢会让 15 秒文案信息不足。
- 音频输入标准化可以做，但不能把后续换音轨当作修复最终结果。

验收指标：

- 输入音频响度接近参考目标，不出现过高峰值。
- 生成结果 `mouth_face_ratio_p90 <= 0.075`。
- 语义完整，不因降爆口丢核心卖点。

blocked 条件：

- 需要本地换音轨才能看起来同步。
- 声音不再像目标声音。
- 文案压缩导致销售信息不可用。

## 方案三：文案节奏侧控制

目标：减少连续大张口音节和高密度信息，让模型口型有缓冲。

做法：

- 优先保留：三层加厚、防爆防压、400 斤、母婴级 PVC、无异味、放心用。
- 减少连续强口型词和过密数字/承诺。
- 每 3-5 秒保留一个自然短停顿。
- 避免在 15 秒内塞满长句。

为什么能压口型：当前 Wan 原始输出全片失败比例不高，说明局部强音节/密集段可能是主要风险；降低音节密度能减少连续大开口。

风险：

- 文案过短会降低卖点密度。
- 停顿过多会显得弱或不直播。

验收指标：

- 15 秒内文案清晰完整。
- `longest_fail_streak < 5`
- 用户人审认为节奏自然，不像机械念稿。

blocked 条件：

- 压节奏后仍无法降低 `mouth_face_ratio_p90`。
- 文案变得不可用或不符合直播销售语气。

## 方案四：参考素材 / 首帧 / 姿态选择

目标：用更低 mouth-open-risk 的首帧和更稳定的正脸姿态降低初始嘴型放大风险。

做法：

- 继续优先选择闭嘴或微闭嘴首帧。
- 避免露齿、侧脸、低头、脸部被手遮挡、强表情帧。
- 保持脸部占画面比例稳定，不让嘴部区域在脸部比例中过大。
- 首帧候选先做只读 landmark 检测，再进模型生成。

为什么能压口型：Wan 报告显示闭嘴/微闭参考帧能降低基础风险；如果首帧已经露齿或张嘴，后续更容易放大。

风险：

- 太静态的姿态可能降低直播感。
- 参考帧过度追求闭嘴，可能让说话动作僵。

验收指标：

- 首帧 `mouth_face_ratio` 低，牙齿不可见或极少。
- 生成结果人脸不漂，嘴部不变形。
- 动作自然度不低于现有 Wan 样片。

blocked 条件：

- 找不到合适闭嘴正脸参考帧。
- 低风险首帧导致人物过僵或明显不像直播。

## 方案五：生成参数 / prompt / negative prompt 控制

目标：把下一轮 probe 固定成可对比实验，避免盲目抽卡。

做法：

- 固定同一首帧、同一音频、同一文案，只改变 mouth-control prompt。
- 候选 A：当前基线 prompt。
- 候选 B：低幅度口播 prompt。
- 候选 C：低幅度口播 + negative prompt。
- 只输出模型原生结果，不做后处理。

为什么能压口型：变量少，能判断 prompt 是否对 mouth amplitude 有真实影响。

风险：

- prompt 控制可能弱。
- 生成波动可能遮盖参数效果。

验收指标：

- 至少一个候选模型原生 `mouth_face_ratio_p90 <= 0.075`。
- 不牺牲声音同步和人脸稳定。
- 不使用本地修嘴。

blocked 条件：

- 生成 API 不支持所需参数。
- 继续 probe 需要本地兜底。
- 预算、权限或 API 状态不允许。

## 下一轮最小 probe

1. 只选 2-3 个 Wan-S2V 原生候选。
2. 不做本地嘴部压缩、局部修嘴、羽化融合、换音轨。
3. 每个候选输出同一套指标：`duration_sec`、`has_audio`、`p90_mouth_open_ratio`、`mouth_face_ratio_p90`、`fail_frame_percent`、`longest_fail_streak`、`face_stability_observation`、`human_review_status`。
4. 先做 15 秒，不扩 30/60 秒。
5. 只有模型原生候选过指标且用户人审可接受，才进入 30 秒 / 60 秒稳定性测试。

## fallback_route

如果 Wan-S2V 原生候选仍口型偏大：

- `fallback_1`：回到 VideoRetalk A 声音克隆路线继续优化。
- `fallback_2`：换有显式 mouth amplitude / expression control 的模型或供应商。
- `fallback_3`：仅在用户本轮明确授权时，才可讨论本地后处理兜底；即便授权，也必须标记 `local_fallback_used=true`，不得写成模型原生能力。

## 禁止事项

- 禁止本地嘴部压缩作为默认主路线。
- 禁止局部修嘴、羽化融合、换音轨、补帧后冒充 Wan-S2V 原生结果。
- 禁止把后处理技术指标通过写成人审通过。
- 禁止把 15 秒通过外推成 30 秒 / 60 秒稳定。
