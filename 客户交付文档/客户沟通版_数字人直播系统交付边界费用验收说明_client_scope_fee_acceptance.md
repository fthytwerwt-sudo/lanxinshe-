# 数字人直播系统与短视频切片工作流合作说明

首期交付边界 | 费用拆分 | 阶段验收 | 后续扩展说明

客户沟通版 | 用于正式推进前确认项目范围、费用、验收、维护和后续扩展方式。

资料依据：数字人直播间完整资料交付清单（重点资料版）、抖音 AI 数字人直播提案（时间节点增强版）、以及本轮已确认的首期合作边界。


## 1. 文档目的说明

这份说明不是为了限制合作，而是为了让后续合作更省心：在正式执行前，把系统边界、交付内容、费用构成、验收标准和后续扩展方式提前说明清楚。这样客户能清楚知道第一期能拿到什么，我方也能按阶段稳定交付。

- 明确首期完成哪些内容，哪些内容建议后续扩展。
- 明确客户可以自行调整的内容，以及需要技术支持的内容。
- 明确 7 万左右预算对应的服务范围，而不是把第三方软件和无限扩展混入首期。
- 明确每个阶段交付什么、如何验收、什么情况属于失败或需要顺延。

## 2. 合作总定位

第一期建议定位为：单人数字人直播 Beta + 短视频切片 MVP。首期先完成一个单人 3D 数字人直播间，目标是 1:1 复刻一个真人主播的讲课、表达、促单、评论回复和异议处理能力，并围绕现有直播内容沉淀可复用的切片结构。

可扩展不等于首期全部实现。首期会按可扩展结构设计，但交付目标会先聚焦在一个可验证、可复跑的单人数字人直播 Beta。

直播时长按双层验收口径理解：20-30 分钟只作为前期链路验证 / 小范围试播，用来确认数字人表现、评论读取、AI 回复、人工接管和日志留存链路是否跑通；首期 Beta 的最终直播交付验收，是在人工中控兜底情况下完成单场不少于 4 小时、目标 4-5 小时的稳定直播验证。该验收不是完全无人直播验收，也不取消人工接管。

| 维度 | 首期定位 | 后续扩展 |
| --- | --- | --- |
| 直播角色 | 一个单人 3D 数字人角色，围绕一个真人主播做能力蒸馏 | 新增导师、医生、专家等角色包 |
| 核心场景 | 讲课场景、带货 / 课程转化场景，口播作为讲课简化版 | 访谈、多角色、多专家、多业务线场景 |
| 切片工作流 | 围绕现有直播沉淀 4-5 类可复用结构模板 | 新增平台尺寸、字幕样式、钩子风格和内容方向 |
| 自动化程度 | Beta 阶段保留人工中控和人工兜底 | 稳定后逐步降低人工接管比例 |

## 4. 针对客户问题的逐项回应

### 4.1 换形象、换主题是否需要重新开发

我们会尽量把可调整的内容做成配置项，但不同类型的调整工作量并不一样。背景、资料、权益和话术属于相对可配置内容；如果更换数字人本人或切换到完全不同的业务主题，则需要重新准备角色资料、声音形象、知识库和话术规则，建议作为后续新增角色或新增主题单独评估。

| 调整类型 | 首期处理方式 | 边界说明 |
| --- | --- | --- |
| 直播背景 | 相对简单，可做成配置 | 更换背景图、贴片、权益区位置属于视觉配置 |
| 课程资料 | 通过资料库 / 课程知识库更新 | 资料事实、价格、权益、退款规则需要客户确认 |
| 标题、活动、卖点 | 可配置或通过话术库调整 | 不能承诺成交、收益或课程效果结果 |
| 同类课程主题 | 资料完整时可适配 | 需要补齐新课程知识库和常见问题 |
| 跨赛道主题 | 需要重新整理知识库和业务规则 | 不建议混入首期免费变更 |
| 更换数字人本人 | 新增角色工程 | 不是简单配置，涉及形象、声音、授权、人设和话术 |

### 4.2 数字人角色扩展：导师、医生、专家

后续可以按“新增角色包”的方式扩展。每个角色都需要单独准备形象素材、声音素材、肖像授权、声音授权、角色人设、语气风格、专业知识库、禁用表达、常见问题、异议处理、合规边界和样片测试。

导师角色可以作为后续优先扩展；医生 / 专家角色涉及更强合规风险，需要额外审核，不建议混入首期。首期会预留结构，但不承诺首期完成多角色管理后台。

### 4.3 直播场景配置：口播、访谈、带货、课程讲解

| 场景类型 | 包含内容 | 首期建议 |
| --- | --- | --- |
| 视觉场景 | 背景图、直播间贴片、课程卡、权益区、主播站位、画面布局 | 讲课和带货场景可配置，背景更换相对简单 |
| 业务场景 | 讲课逻辑、促单逻辑、评论互动、访谈问答、专家咨询、风险边界 | 讲课和带货为重点；访谈、专家、多人成为二期评估项 |
| 口播场景 | 单人讲述、课程介绍、活动说明 | 可作为讲课场景简化版 |
| 多人 / 复杂访谈 | 多角色关系、问答脚本、控场逻辑 | 不含首期 |

### 4.4 短视频切片：固定流程还是持续优化工作流

首期不是只做一条死流程，而是做 4-5 类可复用切片结构模板，AI 根据直播内容自动适配。后续新增风格、尺寸、平台和新内容类型，可以在模板库基础上单独扩展。

| 切片结构 | 建议结构 | 用途 |
| --- | --- | --- |
| 引流型 | 痛点钩子 -> 结果感 -> 继续观看 / 进直播 / 咨询 | 吸引目标用户进入直播或咨询 |
| 课程讲解型 | 问题 -> 原因 -> 方法 -> 步骤 | 把直播讲解整理成可理解短内容 |
| 异议回应型 | 用户质疑 -> 解释 -> 依据 / 案例 -> 引导 | 回应价格、效果、时间等疑虑 |
| 案例信任型 | 案例背景 -> 改变过程 -> 结果反馈 -> 课程承接 | 建立信任，不夸大收益 |
| 促单转化型 | 权益 -> 适合人群 -> 风险解除 -> 下单提醒 | 合规推动转化 |

## 6. 首期不包含哪些内容

以下内容并不是不能做，而是建议作为第二阶段或后续优化需求单独评估。这样第一期可以先聚焦在最关键的单人数字人直播和切片 MVP 上，确保项目能够真实跑通。

- 多数字人角色自由切换后台、多人同场直播、导师 / 医生 / 专家角色完整落地。
- 复杂访谈场景、完全无人直播、长期直播代运营。
- 无限新增切片风格、无限平台尺寸适配、新增 CRM / 数据看板 / 自动私信回复等系统。
- 第三方软件费用、第三方 API 费用、云服务费用、数字人工具订阅费。
- 抖音平台规则变化导致的额外适配，或客户后续新增业务线导致的系统重构。

## 11. 每阶段验收标准与失败标准

| 验收对象 | 通过标准 | 失败标准 | 处理方式 |
| --- | --- | --- | --- |
| 资料锁定 | 资料清单、授权、价格权益、退款规则、首期范围均确认 | 资料长期不完整，授权不明确，仍要求首期无限扩展 | 顺延、暂停或按已有资料降级 |
| 数字人样板 | 形象不出戏，声音语速可接受，讲课和促单表达准确合规 | 形象完全不可接受、声音明显不符、课程表达严重错误 | 回到样片优化或评估替代工具 |
| 平台链路 | 评论读取、AI 回复延迟、人工接管和日志留存可验证 | 评论读取不稳定、回复延迟过高、平台规则限制自动化 | 离线模拟、人工中控、降级直播流程 |
| AI 回复质量 | 能围绕课程事实回复，不乱承诺，不夸大，不跑题，高风险问题可接管 | 频繁跑题、错报权益、乱承诺效果、无日志 | 补资料、修规则、加禁区词和接管条件 |
| Beta 稳定直播 | 在人工中控兜底情况下，单场不少于 4 小时、目标 4-5 小时，覆盖讲课、互动、评论回复、异议处理、促单引导、人工接管和日志复盘 | 未完成 4 小时以上直播验证，人工兜底不可执行，高风险问题未接管，AI 频繁错答价格、权益、退款、效果承诺，或无直播录屏、评论日志、AI 回复日志、人工接管记录 | 补资料、修规则、强化接管条件，必要时暂停、顺延或采用替代链路 |
| 短视频切片 | 输出教学样片和 3-5 条直播切片，能归类到结构模板，可独立发布 | 结构混乱、字幕标题不可用、无法复跑 | 回到模板和判断规则优化 |
| 合作推进 | 阶段确认及时，阶段款按约定支付，新增需求走变更评估 | 长期不确认、中途大改目标、阶段款未支付 | 暂停、顺延或按已完成阶段结算 |

公平边界：客户资料未确认、账号权限异常、抖音平台限制、第三方软件故障、客户临时改口径等外部因素，不直接算作我方交付失败，应先进入暂停、顺延或替代方案评估。

最终 Beta 验收证据必须包括：直播录屏、评论日志、AI 回复日志、人工接管记录、问题清单和复盘记录。失败判断包括：AI 频繁错答价格、权益、退款、效果承诺，高风险问题未接管，或人工兜底不可执行。20-30 分钟链路验证只证明前期链路跑通，不代表最终交付直播时长。

- AI 频繁错答价格、权益、退款、效果承诺。
- 高风险问题未接管。
- 人工兜底不可执行。

## 12. 维护范围

### 12.1 首期包含的基础维护

建议交付确认后提供 15-30 天基础维护，覆盖首期范围内 bug 修复、配置文档错误修正、基础使用答疑、已交付流程跑不通的排查、简单参数调整和 SOP 使用解释。

### 12.2 不包含在免费维护内

- 长期代运营直播、每场直播陪跑调试、新增数字人角色、新增导师 / 医生 / 专家角色。
- 新增访谈场景、新切片风格、新平台尺寸适配、第三方软件接口变化、抖音规则变化。
- 客户账号权限异常、临时更换课程体系、新增业务线、系统部署迁移、长期数据看板和 CRM 接入。

## 13. 需求变更机制

不影响首期目标的小调整，可以放入当前阶段优化，不单独收费，前提是工作量可控。影响范围、周期或成本的新增需求，进入变更评估，单独确认报价和排期，并不影响原定首期交付验收。

| 典型变更 | 建议处理方式 |
| --- | --- |
| 新增数字人、导师、医生、专家 | 新增角色包评估，单独确认资料、授权、合规和样片测试 |
| 新增访谈、多角色同场、新平台适配 | 单独评估场景逻辑、链路、排期和费用 |
| 新增私信自动回复、CRM、后台配置系统、数据看板 | 作为新系统模块单独评估，不混入首期 |
| 新增长期运营服务 | 按运营频次、人力投入和响应时效另行报价 |


## 14. 客户资料交付责任

资料交付清单的核心原则是：客户提供原始资料和核心业务事实，我方负责后续解析整理。客户不需要提前完成数据清洗、评论分类、直播节奏拆解或成交节点分析。

| 资料模块 | 建议提交内容 | 建议数量 / 格式 | 用途 |
| --- | --- | --- | --- |
| 完整直播录屏 | 自家历史直播、对标直播间录屏、每场直播说明表 | 20 场录屏，MP4/MOV + 表格 | 分析直播节奏、互动方式、成交节点和表达风格 |
| 课程 / 产品资料 | 价格、权益、交付、退款、适合人群、禁用表达、客服标准解释 | 每个课程 / 产品 1 个文件夹 | 确保讲解内容、权益表达和风险边界准确 |
| 主播话术与成交资料 | 直播脚本、提词器、成交话术、客服话术、社群转化话术、成交 SOP | 全部现有版本 | 支撑讲课表达、成交引导和话术迁移 |
| 用户问题与异议资料 | 高频问题、私信咨询、客服记录、销售咨询、售后问题、负面质疑 | 合计约 2000 条 / 份记录 | 支撑问题识别、异议处理和人工接管触发 |

- 客户需要保证课程、价格、权益、退款规则真实准确。

- 客户需要对 AI 话术中的业务事实和高风险表达给出确认意见。

## 15. 我方交付责任

- 根据资料进行主播能力蒸馏，整理讲课结构、痛点、异议、促单规则。
- 搭建 AI 直播大脑 v0、单人数字人直播样板、讲课和带货场景初版。

- 完成教学视频自动流 MVP 和直播切片自动流 MVP。

## 16. 合规与风险边界

- 不做虚假承诺，不承诺成交结果，不承诺课程效果，不做虚假稀缺，不夸大收益。
- 医生 / 专家类内容需额外合规审核，不建议混入首期。
- AI 话术需要客户确认业务事实；高风险问题进入人工接管。
- 平台政策变化、第三方工具能力限制和账号权限问题属于外部风险，需要按实际情况降级或调整。

## 17. 成果归属与资料安全

- 客户提供的课程资料、录屏、主播形象、声音和话术归客户所有，未经授权不得用于其他客户。
- 我方通用方法、通用工具、通用流程模板不等于全部源码买断。
- 为客户定制的配置、知识库、话术库、规则文件可交付客户使用。
- 是否交付源码、账号、部署环境，需要另行约定。
- 敏感信息需脱敏交付；API key、密码、验证码通过安全方式交接。

## 18. 暂停、顺延与结算机制

| 可暂停或顺延的情况 | 建议处理 |
| --- | --- |
| 客户资料未按时提供、阶段产物未确认、数字人授权不明确 | 项目顺延，不计为我方延期 |
| 第三方软件不可用、账号权限不完整、平台规则变化 | 先降级测试或调整链路，再评估影响 |
| 客户新增需求影响原计划、阶段款未支付 | 暂停当前阶段，重新确认范围、排期和费用 |
| 核心链路失败 | 按已完成产物和阶段结算，双方重新评估替代方案 |

结算建议：已完成阶段不退款，未开始阶段可协商；中途新增需求另行报价；核心链路失败时按已完成产物和阶段结算。
